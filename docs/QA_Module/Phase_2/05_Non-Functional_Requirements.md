# 05. Non-Functional Requirements (ความต้องการที่ไม่ใช่ฟังก์ชัน)

เอกสารฉบับนี้กำหนดเกณฑ์มาตรฐานทางด้านประสิทธิภาพ ความปลอดภัย และความน่าเชื่อถือของระบบ SAMS Module QA เพื่อเป็นตัวชี้วัดความสำเร็จในเชิงเทคนิคที่นอกเหนือจากการทำงานพื้นฐาน

## 1. Performance & Responsiveness (ประสิทธิภาพและความเร็ว)
- **Dashboard Load Time:** หน้า Dashboard สรุปผลรวม และตาราง Matrix จะต้องโหลดข้อมูลขึ้นมาแสดงผลพร้อมให้โต้ตอบได้ ภายในเวลา **น้อยกว่า 3 วินาที** (ในสภาวะการเชื่อมต่ออินเทอร์เน็ตปกติ)
- **Data Rendering:** ตาราง QA Training Matrix และ Authorization Matrix ซึ่งอาจมีจำนวนข้อมูลพนักงานมากกว่า 100 คน และมีคอลัมน์เกิน 20 คอลัมน์ จะต้องสามารถเลื่อน (Scroll) ทั้งแนวตั้งและแนวนอนได้อย่างลื่นไหล ไม่เกิดอาการค้าง (Lag) โดยใช้เทคนิค Virtualization หรือ Pagination 
- **Export Speed:** การสั่ง Export ไฟล์ Excel สำหรับข้อมูลพนักงาน 500 คน ต้องใช้เวลาประมวลผลไม่เกิน 10 วินาที

## 2. Security & Compliance (ความปลอดภัยและมาตรฐาน)
- **Authentication & Session:** การเข้าสู่ระบบต้องใช้กระบวนการเข้ารหัส (เช่น JWT) โดย Token จะถูกเก็บใน `httpOnly` Cookies เพื่อป้องกันการถูกขโมยผ่านสคริปต์ (XSS Protection)
- **Role-Based Access Control (RBAC):** ระบบจะต้องป้องกันการเข้าถึง API และ UI ทุกจุด หากผู้ใช้งานพยายามเข้าถึง URL นอกเหนือจากสิทธิ์ที่ตนมี ระบบต้องบล็อก (Forbidden 403) และ Redirect กลับไปหน้า Home ทันที
- **Data Auditing:** ทุกการแก้ไขสถานะในส่วนของ Training Record และ Authorization จะต้องมี Audit Trail บันทึกในระบบ Backend อย่างชัดเจนว่าใคร (User ID) เป็นผู้แก้ แก้เมื่อไหร่ (UTC Time) และข้อมูลเดิมคืออะไร (Before/After)
- **GDPR / PDPA Compliance:** ข้อมูลส่วนบุคคล (PII) เช่น เลขบัตรประชาชนของพนักงาน หรือข้อมูลการติดต่อ จะถูกจำกัดสิทธิ์การมองเห็นเฉพาะ HR และ QA Manager เท่านั้น

## 3. Availability & Reliability (ความเสถียรภาพ)
- **System Uptime:** ระบบออกแบบมาเพื่อให้มีความพร้อมใช้งานสูง ไม่ต่ำกว่า 99.5% 
- **Timezone Integrity:** โครงสร้างข้อมูลเวลาต้องรองรับการเก็บเป็น UTC ในระดับ Database อย่างสมบูรณ์แบบ เพื่อให้สามารถรองรับการขยายตัวของสายการบินที่ปฏิบัติงานใน Timezone อื่นๆ ได้ในอนาคต

## 4. Usability & Accessibility (การใช้งานและประสบการณ์ผู้ใช้)
- **Visual Indicators:** ในหน้า Monitoring Matrix สถานะสีแดง (หมดอายุ) และสีเหลือง (ใกล้หมดอายุ) จะต้องมาพร้อมกับ "ป้ายกำกับข้อความ (Text Label)" หรือไอคอนเตือนเสมอ เพื่อช่วยเหลือผู้ใช้งานที่มีภาวะตาบอดสี (Color-blind friendly)
- **Responsive Layout:** 
  - ระบบรองรับการทำงานเต็มรูปแบบ (Full Feature) บน Desktop และ Laptop (หน้าจอความละเอียดกว้างกว่า 1024px)
  - สำหรับหน้าจอ Tablet (เช่น iPad) ระบบจะยุบแถบเมนู (Sidebar) อัตโนมัติ เพื่อเพิ่มพื้นที่ในการดู Matrix
  - *ข้อจำกัด:* ไม่สนับสนุนการแก้ไข Matrix บนหน้าจอสมาร์ทโฟน (Mobile) เนื่องจากหน้าจอมีขนาดเล็กเกินไป

## 5. Scalability (ความสามารถในการขยายระบบ)
- Database Schema ในส่วนของ Authorization ถูกออกแบบโดยใช้ Join Tables ทำให้สามารถรองรับการเพิ่มสายการบิน (Customer) หรือหน่วยงานตรวจสอบ (Authority) ใหม่ๆ ได้แบบไดนามิก (Dynamic) โดยไม่จำเป็นต้องปรับแก้โครงสร้าง Database หรือโค้ดในระดับ Core Logic
