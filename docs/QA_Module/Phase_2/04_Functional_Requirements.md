# 04. Functional Requirements (ความต้องการทางฟังก์ชัน)

เอกสารฉบับนี้ระบุถึงความสามารถหลัก (Core Features) ของระบบ SAMS Module QA โดยแยกตาม Module ย่อย และอธิบายในรูปแบบของ Input -> Process -> Output เพื่อให้ง่ายต่อการนำไปพัฒนาและทดสอบ

---

## 1. QA Staff List Management (การจัดการรายชื่อพนักงาน QA)
**คำอธิบาย:** ระบบสำหรับดูและจัดการรายชื่อพนักงานทั้งหมดที่อยู่ในความดูแลของ QA
- **Input:** ผู้ใช้กรอกข้อมูลพนักงานใหม่ หรือทำการค้นหา (Search) / กรอง (Filter) พนักงานจากรายชื่อ
- **Process:** 
  - ระบบบันทึกข้อมูลพนักงานลง Database 
  - ระบบตรวจสอบความซ้ำซ้อนของข้อมูล (เช่น Employee ID ต้องไม่ซ้ำ)
  - ระบบดึงข้อมูลพนักงานจาก Backend มาแสดงผลบนตาราง
- **Output:** ตารางแสดงรายชื่อพนักงาน (Data Table) พร้อมปุ่มกดดู Profile เชิงลึก และสถานะโดยรวม

## 2. QA Training Matrix Monitoring (การติดตามสถานะการอบรม)
**คำอธิบาย:** หน้า Dashboard สรุปสถานะการอบรมของพนักงานทั้งหมดในรูปแบบ Matrix
- **Input:** ผู้ใช้เลือกช่วงเวลา หรือ Filter เฉพาะกลุ่มพนักงาน / หมวดหมู่หลักสูตร (เช่น INI, R2Y)
- **Process:**
  - ระบบคำนวณวันหมดอายุ (Expiry Date) โดยอ้างอิงจากวันที่เข้าอบรมล่าสุด (Last Trained Date) และรอบระยะเวลาของหลักสูตร (Valid Period)
  - จัดกลุ่มสถานะตามสี: 
    - สีเขียว = ปกติ (Valid)
    - สีเหลือง = ใกล้หมดอายุ (Expiring within X days)
    - สีแดง = หมดอายุ (Expired)
- **Output:** ตาราง Training Matrix ที่มี Cell สัญลักษณ์สีต่างๆ พร้อม Tooltip แสดงรายละเอียดวันที่เมื่อนำเมาส์ไปชี้ (Hover)

## 3. Course Management & Training Scheduler (การจัดการหลักสูตรและตารางสอน)
**คำอธิบาย:** ระบบสำหรับการตั้งค่าหลักสูตร และการวางแผนคลาสเรียน
- **Input:** 
  - *Course Management:* ผู้ใช้สร้างหลักสูตรใหม่ (ชื่อ, รหัส, ระยะเวลา Validity)
  - *Training Scheduler:* ผู้ใช้สร้าง Session คลาสเรียน (วันที่, เวลา, ผู้สอน) และเลือกพนักงาน (Attendees) เข้าเรียน
- **Process:**
  - ระบบตรวจสอบว่าผู้เข้าเรียนใน Scheduler มีตารางซ้อนทับกันหรือไม่ (Conflict checking)
  - เมื่อการอบรมเสร็จสิ้น และอัปเดตสถานะเป็น "Passed" ระบบจะไปอัปเดตวันที่เรียนล่าสุดลงใน Profile ของพนักงานแต่ละคนโดยอัตโนมัติ
- **Output:** 
  - หน้าปฏิทิน (Calendar View) หรือ Timeline (Gantt Chart View) แสดงตารางการอบรม
  - ประวัติการเข้าอบรมในหน้า Staff Profile ถูกอัปเดต

## 4. Authorization Monitoring (การติดตามการอนุมัติสิทธิ์)
**คำอธิบาย:** หน้า Matrix สำหรับตรวจสอบและแก้ไขสิทธิ์ของช่างซ่อมบำรุง แบ่งเป็น 2 ส่วนคือ Customer (สายการบินลูกค้า) และ Authority (หน่วยงานกำกับดูแล)
- **Input:** ผู้ใช้ (QA) กดคลิกที่ Cell ในหน้า Matrix เพื่อเปิด Modal "Edit Authorization" จากนั้นเลือกสายการบิน (Customer), เครื่องบิน (Aircraft Type) และกำหนดวันที่ Issue / Expire
- **Process:**
  - ระบบบันทึกประวัติการแก้ไข (Update Status) และข้อมูลวันที่ลงในตาราง Authorization ของพนักงานคนนั้น
  - ระบบทำการเช็ค (Dual Check) ว่าพนักงานคนดังกล่าวมีสิทธิ์ (Eligibility) ในการเซ็นใบรับรอง (CRS) ครบถ้วนตามเงื่อนไขหรือไม่
- **Output:** 
  - Matrix แถวรายชื่อพนักงาน คอลัมน์สายการบิน/Authority จะแสดง Icon อนุมัติสีเขียว หรือสัญลักษณ์ตามที่กำหนด
  - ข้อมูล Audit Trail ใน Backend ระบุว่าใครเป็นผู้อนุมัติ

## 5. Report & Data Export (การออกรายงานและส่งออกข้อมูล)
**คำอธิบาย:** ฟังก์ชันการนำข้อมูลใน Matrix ออกมาในรูปแบบ Excel 
- **Input:** ผู้ใช้คลิกปุ่ม "Export to Excel" ในหน้า Monitoring หรือ Authorization
- **Process:**
  - ระบบรับคำสั่งส่งไปยัง Backend (หรือจัดการบน Frontend ผ่าน SheetJS/XLSX)
  - จัดเรียงข้อมูล Column ตามที่ปรากฏบนหน้าจอ และสร้าง Multiple Sheets (หากจำเป็น เช่น แยกตามสายการบิน หรือแผนก)
- **Output:** ไฟล์ `.xlsx` ที่โหลดลงเครื่องผู้ใช้ ซึ่งสามารถเปิดอ่านและนำไปใช้ทำ Report นอกระบบได้ทันที
