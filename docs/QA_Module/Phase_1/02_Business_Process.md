# 02. Business Process (กระบวนการทางธุรกิจ)

เอกสารส่วนนี้เปรียบเทียบกระบวนการทำงานในรูปแบบเดิม (As-Is) และรูปแบบใหม่หลังจากนำระบบ SAMS Module QA มาใช้ (To-Be) เพื่อให้เห็นถึง Pain Point และแนวทางการแก้ไขที่ชัดเจน

## 1. As-Is Process (กระบวนการทำงานในปัจจุบัน)
ปัจจุบัน การจัดการข้อมูลประวัติการอบรม (Training Records) และการอนุมัติสิทธิ์ (Authorization) ดำเนินการผ่านรูปแบบเอกสารและโปรแกรมตารางคำนวณ (Spreadsheet) เป็นหลัก 

**Flow การทำงานเดิม:**
1. **เก็บข้อมูลบุคลากร:** ข้อมูลพนักงานและ Certificate ต่างๆ (เช่น SAMS-FM-CM-008, SAMS-FM-CM-036) ถูกเก็บแยกเป็นไฟล์ PDF หรือ Excel ของแต่ละคน
2. **บันทึกการฝึกอบรม:** เมื่อมีการฝึกอบรม Trainer จะให้เซ็นชื่อใน Attendance Sheet (เช่น SAMS-FM-CM-026) และ QA จะต้องนำข้อมูลมาอัปเดตลงในไฟล์ Master Excel (เช่น SAMS-FM-CM-072) ด้วยตนเอง
3. **ติดตามวันหมดอายุ:** QA ต้องเปิดดูไฟล์ Master Excel เป็นระยะ เพื่อค้นหาพนักงานที่ Certificate หรือ Authorization ใกล้หมดอายุ (ใช้วิธี Filter หรือดูจากสีที่ทำไว้เอง)
4. **ตรวจสอบสิทธิ์ก่อนปฏิบัติงาน:** เมื่อต้องการตรวจสอบว่าพนักงานคนนี้มีสิทธิ์เซ็น Release เครื่องบินสายการบิน A หรือไม่ ต้องไปค้นหาในเอกสาร List of Customer Authorization (PDF)

**Pain Points (ปัญหาที่พบ):**
- **ความล่าช้าและซ้ำซ้อน (Inefficiency):** ข้อมูลกระจัดกระจายหลายไฟล์ ต้องกรอกข้อมูลซ้ำซ้อน
- **ความเสี่ยงตกหล่น (Human Error):** มีโอกาสลืมอัปเดตวันหมดอายุ ทำให้บุคลากรปฏิบัติงานทั้งที่สิทธิ์ขาดอายุ (Non-compliance risk)
- **ตรวจสอบยาก (Lack of Visibility):** ไม่สามารถเห็นภาพรวมของแผนกได้ทันที การเตรียมเอกสารให้ Auditor ใช้เวลานาน

## 2. To-Be Process (กระบวนการทำงานในระบบใหม่)
ระบบ SAMS Module QA จะเข้ามาแทนที่กระบวนการ Manual โดยรวมศูนย์ข้อมูลไว้ใน Web Application

**Flow การทำงานใหม่:**
1. **ศูนย์รวมข้อมูล (Centralized Database):** ข้อมูล Staff, ข้อมูล Aircraft Type, Authority Code ถูกดึงจาก Master Data ของระบบมาเชื่อมโยงกัน
2. **จัดการการฝึกอบรม (Training Management):**
   - QA จัดตารางเรียนใน `Training Scheduler`
   - ระบบอัปเดตสถานะใน `QA Training Matrix` ให้อัตโนมัติเมื่อพนักงานผ่านการประเมิน
3. **การแจ้งเตือนและการตรวจสอบ (Automated Monitoring):**
   - ระบบ `Monitoring` และ `Authorization Matrix` จะแสดงสถานะเป็นสี (เช่น เขียว=ปกติ, เหลือง=ใกล้หมดอายุ, แดง=หมดอายุ)
   - หน้าจอสรุป (Dashboard) ทำให้ QA Manager เห็นได้ทันทีว่าใครต้องเข้าอบรมเพิ่มเติมหรือต่ออายุ Authorization
4. **การตรวจสอบสิทธิ์รวดเร็ว (Instant Verification):** สามารถค้นหาชื่อพนักงานในระบบและดู Profile เพื่อตรวจสอบ Authorization ของสายการบิน (Customer) และหน่วยงาน (Authority) ได้ภายในไม่กี่คลิก

## 3. สรุปความเปลี่ยนแปลง (Impact & Benefits)
| หัวข้อ | As-Is (แบบเดิม) | To-Be (แบบใหม่) |
| :--- | :--- | :--- |
| **Data Storage** | ไฟล์ Excel, PDF แยกตามโฟลเดอร์ | Database รวมศูนย์ ค้นหาง่าย |
| **Expiry Tracking** | ต้องไล่ดูทีละบรรทัดใน Excel | ดูจากสีใน Matrix (Visual Indicator) |
| **Reporting / Auditing** | ใช้เวลาเตรียมการหลายวัน | เข้าหน้าจอ Matrix แล้วสั่ง Export / ดูได้ทันที |
| **Data Integrity** | อาจมีการพิมพ์ผิดหรือลืมบันทึก (Typo) | มีการ Validate ข้อมูลจาก Master Data |
