## 1. ภาพรวมของระบบ

**SAM Airline Maintenance** (SAMS) เป็นระบบเว็บแอปพลิเคชันสำหรับบันทึก ติดตาม และจัดการข้อมูลการบำรุงรักษาอากาศยาน (Line Maintenance) ออกแบบมาเพื่อใช้งานภายในสนามบิน โดยมีฟังก์ชันหลักดังนี้:

| โมดูล | คำอธิบาย |
|-------|----------|
| **Dashboard** | แสดงภาพรวมสถิติการทำงาน กราฟ และข้อมูลสรุป |
| **Flight List** | จัดการรายการเที่ยวบินทั้งหมด สร้าง/แก้ไข/ค้นหา |
| **THF Form** | แบบฟอร์ม Technical Handling Form สำหรับบันทึกการบำรุงรักษา (5 ขั้นตอน) |
| **Contract** | จัดการสัญญาบริการ |
| **Invoice** | จัดการใบแจ้งหนี้ |
| **Report** | สร้างรายงานต่างๆ |
| **Master Data** | จัดการข้อมูลพื้นฐาน (สายการบิน, พนักงาน, สถานี, บัญชีผู้ใช้, บทบาท, สิทธิ์) |

## 3. การเข้าสู่ระบบ (Login)

### 📷 หน้าจอ: Login Page

> **หมายเหตุ:** แนบภาพหน้าจอ Login Page ที่นี่
> ![Login Page](screenshots/login_page.png)

### คำอธิบายหน้าจอ

หน้า Login ประกอบด้วย:
- **โลโก้ระบบ** แสดงอยู่ตรงกลางด้านบนของฟอร์ม
- **ข้อความ** "Sign in to your account to start using"
- **ฟอร์มเข้าสู่ระบบ** อยู่ภายในกรอบสีขาวพร้อมเงา

### ขั้นตอนการเข้าสู่ระบบ

1. เปิด Web Browser แล้วเข้าไปที่ URL ของระบบ
2. ระบบจะแสดงหน้า **Login** โดยอัตโนมัติ
3. กรอกข้อมูลในฟอร์ม:

| ฟิลด์ | คำอธิบาย | ตัวอย่าง |
|-------|----------|----------|
| **Email** | อีเมลที่ลงทะเบียนไว้ในระบบ | `user@example.com` |
| **Password** | รหัสผ่าน (อย่างน้อย 4 ตัวอักษร) | `••••••••` |

4. คลิกปุ่ม **"Sign In"** (ปุ่มสีน้ำเงิน)
5. รอระบบตรวจสอบข้อมูล — จะแสดงสถานะ "Loading..."
6. เมื่อเข้าสู่ระบบสำเร็จ จะเปลี่ยนหน้าไปยัง **Dashboard** โดยอัตโนมัติ

### เคล็ดลับ

- 🔒 คลิกไอคอน **👁** ข้างช่องรหัสผ่านเพื่อแสดง/ซ่อนรหัสผ่าน
- ⚠️ หากกรอกข้อมูลไม่ถูกต้อง ระบบจะแสดงข้อความแจ้งเตือนสีแดงใต้ช่องที่ผิดพลาด
- 🔄 หากลืมรหัสผ่าน กรุณาติดต่อผู้ดูแลระบบ

| คอลัมน์ | คำอธิบาย |

2. ระบบจะค้นหาและแสดงผลลัพธ์โดยอัตโนมัติ
3. เมื่อเปลี่ยนตัวกรอง ระบบจะรีเซ็ตกลับไปหน้า 1

### 6.3 การสร้างเที่ยวบินใหม่ (Create Flight)

### 📷 หน้าจอ: Create Flight Dialog

> ![Create Flight](screenshots/create_flight_dialog.png)

### ขั้นตอนการสร้างเที่ยวบินใหม่

#### ข้อมูลที่ต้องกรอก

**ส่วนข้อมูลหลัก:**

| ฟิลด์ | ประเภท | จำเป็น | คำอธิบาย |
|-------|--------|--------|----------|

| **Date** | Date Picker | ✅ | วันที่มาถึง (รูปแบบ DD/MM/YYYY) |

**ส่วนข้อมูล Departure (ขาออก — UTC Time):**

| ฟิลด์ | ประเภท | จำเป็น | คำอธิบาย |

| **ATD (UTC)** | Time Input | ❌ | เวลาออกเดินทางจริง |

> ⚠️ **หมายเหตุ:** หากกรอกข้อมูล Departure ฟิลด์ใดฟิลด์หนึ่ง จะต้องกรอก Date และ STD ให้ครบ

**ส่วนข้อมูลเพิ่มเติม:**

| ฟิลด์ | ประเภท | คำอธิบาย |
|-------|--------|----------|
| **Bay** | Text Input | หมายเลขเบย์ |
| **THF Number** | Text Input | หมายเลข THF |
| **Status** | Dropdown | สถานะ (ค่าเริ่มต้น: Normal) |
| **Maintenance Status** | Dropdown | สถานะการบำรุงรักษา |
| **Note** | Textarea | หมายเหตุ |

**ส่วน Personnel (บุคลากร):**

| ฟิลด์ | คำอธิบาย |
|-------|----------|
| **CS (Customer Service)** | เลือกพนักงานบริการลูกค้า (เลือกได้หลายคน) |
| **Mechanic** | เลือกช่างซ่อมบำรุง (เลือกได้หลายคน) |

3. ตรวจสอบข้อมูลให้ถูกต้อง
4. คลิก **"Create"** เพื่อบันทึก หรือ **"Cancel"** เพื่อยกเลิก
5. เมื่อบันทึกสำเร็จ จะแสดง Toast message "Flight created successfully"

### 6.4 การแก้ไขเที่ยวบิน (Edit Flight)

### 📷 หน้าจอ: Edit Flight Dialog

> ![Edit Flight](screenshots/edit_flight_dialog.png)

### ขั้นตอนการแก้ไขเที่ยวบิน

1. ในหน้า Flight List คลิกปุ่ม **"Edit"** (✏️) ที่คอลัมน์ Actions ของเที่ยวบินที่ต้องการแก้ไข
2. ระบบจะเปิด **หน้าต่างยืนยัน (Confirm Edit Flight)** — ต้องยืนยันตัวตนก่อนแก้ไข
3. กรอกข้อมูลผู้แก้ไข แล้วคลิก **"Confirm"**
4. ระบบจะแสดงฟอร์มแก้ไขพร้อมข้อมูลปัจจุบันที่บรรจุอยู่แล้ว
5. แก้ไขข้อมูลตามต้องการ (ฟิลด์เดียวกันกับการสร้างเที่ยวบิน)
6. คลิก **"Update"** เพื่อบันทึกการเปลี่ยนแปลง

> 💡 ข้อมูลทั้งหมดรวมถึง Maintenance Status, Personnel (CS/Mechanic) สามารถแก้ไขได้จากหน้านี้

## 7. โมดูล THF Form

**THF (Technical Handling Form)** คือแบบฟอร์มหลักที่ใช้บันทึกรายละเอียดการบำรุงรักษาอากาศยานทั้งหมด ประกอบด้วย **5 ขั้นตอน (Steps)**

### 📷 หน้าจอ: THF Form — Stepper Header

> ![THF Stepper](screenshots/thf_stepper.png)

### การเข้าถึง THF Form

1. ไปที่หน้า **Flight List**
2. คลิกปุ่ม **"THF"** หรือ **"View"** ที่คอลัมน์ Actions ของเที่ยวบินที่ต้องการ
3. ระบบจะเปิดหน้า THF Form พร้อม **Stepper** แสดงขั้นตอนด้านบน

### ส่วน Stepper Header

Stepper แสดงขั้นตอนทั้ง 5 ดังนี้:

```
 ① Flight → ② Services → ③ Equipment → ④ Parts & Tools → ⑤ Attach File
```

- **วงกลมตัวเลข**: แสดงหมายเลขขั้นตอน
- **สีเข้ม**: ขั้นตอนที่กำลังทำอยู่
- **✓ (เครื่องหมายถูก)**: ขั้นตอนที่เสร็จแล้ว
- **สีจาง**: ขั้นตอนที่ยังไม่ถึง
- คลิกที่ตัวเลขเพื่อข้ามไปยังขั้นตอนนั้นๆ ได้

### 7.1 Step 1: Flight (ข้อมูลเที่ยวบิน)

### 📷 หน้าจอ: THF — Step 1: Flight

> ![THF Step 1](screenshots/thf_step1_flight.png)

### คำอธิบาย

ขั้นตอนนี้แสดงและแก้ไขข้อมูลพื้นฐานของเที่ยวบิน ฟิลด์เดียวกับการสร้าง/แก้ไขเที่ยวบินในหน้า Flight List:

- ข้อมูลสายการบิน, สถานี, ทะเบียนอากาศยาน
- ข้อมูล Arrival / Departure (Flight No, Date, STA/ATA, STD/ATD)
- เส้นทาง (Route From / Route To)
- Bay, สถานะ, หมายเหตุ
- บุคลากร (CS / Mechanic)

### การดำเนินการ

1. ตรวจสอบ/แก้ไขข้อมูลที่แสดง
2. คลิก **"Save & Next"** หรือ **"Next"** เพื่อไปยัง Step 2

### 7.2 Step 2: Services (บริการ)

### 📷 หน้าจอ: THF — Step 2: Services

> ![THF Step 2](screenshots/thf_step2_services.png)

### คำอธิบาย

ขั้นตอนนี้ใช้บันทึกรายละเอียดบริการที่ให้กับอากาศยาน โดยแบ่งเป็นหลายส่วน:

#### รายการบริการที่บันทึกได้

| หมวดหมู่ | รายละเอียด |
|----------|------------|
| **Fluid Services (ของเหลว)** | บันทึกข้อมูลน้ำมันเครื่อง (Engine Oil), น้ำมันไฮดรอลิก, สารเติม ฯลฯ |
| **Engine Oil** | เลือก Left Engine หรือ Right Engine พร้อมกรอกปริมาณ |
| **General Services** | บริการทั่วไป เช่น การทำความสะอาด, การตรวจสอบ |
| **Aircraft Towing** | ข้อมูลการลากจูงอากาศยาน — สามารถเพิ่มรายการได้หลายรายการ พร้อมกรอก On Time / Off Time |

#### ส่วน Engine Oil Input

- เลือก Engine: **Left Engine** หรือ **Right Engine** จาก Selector
- กรอกปริมาณน้ำมัน
- ค่าจะถูก apply ไปที่ Engine ที่เลือก; Engine ที่ไม่ได้เลือกมีค่าเท่ากับ 0

### การดำเนินการ

1. กรอกข้อมูลบริการตามจริง
2. คลิก **"Save & Next"** เพื่อบันทึกและไปยัง Step 3
3. หรือคลิก **"Back"** เพื่อกลับไป Step 1

## 12. การออกจากระบบ (Logout)

### ขั้นตอนการออกจากระบบ

1. คลิกที่ **ชื่อผู้ใช้** หรือ **ไอคอนผู้ใช้** ที่มุมขวาบนของ Header
2. คลิก **"Logout"** หรือ **"ออกจากระบบ"**
3. ระบบจะ:
   - ล้างข้อมูล Session ออกจาก Browser
   - ล้างข้อมูลออกจาก Redux State
   - ลบ Token ออกจาก localStorage
   - เปลี่ยนหน้าไปยังหน้า **Login** โดยอัตโนมัติ

> ⚠️ หากไม่ได้ Logout แต่ปิด Browser ทิ้งไว้ ระบบจะจดจำสถานะ Login ไว้ (Auto Restore) — หากเปิดใหม่จะสามารถใช้งานต่อได้โดยไม่ต้อง Login ใหม่

## 13. คำถามที่พบบ่อย (FAQ)

### ❓ ลืมรหัสผ่าน ทำอย่างไร?

ติดต่อผู้ดูแลระบบ (Admin) เพื่อรีเซ็ตรหัสผ่านให้ ระบบไม่มีฟีเจอร์ Forgot Password แบบ Self-Service ในขณะนี้

---

### ❓ ข้อมูลจากหน้าจอไม่แสดง / แสดงช้า

- ตรวจสอบการเชื่อมต่ออินเทอร์เน็ต
- ลอง Refresh หน้าจอ (กด F5 หรือ Ctrl+R)
- หากยังมีปัญหา อาจเป็นเพราะ API Server มีปัญหา — ติดต่อทีมเทคนิค

---

### ❓ เห็นข้อความ "⚠️ Using offline data due to API connection issue"

ระบบไม่สามารถเชื่อมต่อ API ได้ จึงใช้ข้อมูลสำรอง (Fallback Data) ข้อมูลอาจไม่เป็นปัจจุบัน — ลองรอสักครู่แล้ว Refresh

---

### ❓ เปิดหน้า Flight List แล้วไม่มีข้อมูล

- ตรวจสอบ **ช่วงวันที่ (Date Range)** ในตัวกรอง — ค่าเริ่มต้นคือวันนี้ อาจไม่มีเที่ยวบินในวันนั้น
- ลองขยายช่วงวันที่ให้กว้างขึ้น
- ตรวจสอบว่าไม่ได้กรองด้วย Station หรือ Airline ที่ไม่มีข้อมูล

---

### ❓ กรอกข้อมูลในฟอร์มแล้วมีขอบแดง

ฟิลด์ที่มีขอบแดงหมายถึงข้อมูลไม่ถูกต้องตาม Validation:
- ตรวจสอบรูปแบบวันที่: ต้องเป็น **DD/MM/YYYY**
- ตรวจสอบรูปแบบเวลา: ต้องเป็น **HH:mm**
- ฟิลด์ที่มีเครื่องหมาย * หรือ "Required" ต้องกรอกข้อมูล

---

### ❓ ไม่สามารถแก้ไขเที่ยวบินได้

- ต้องกดยืนยันตัวตน (Confirm) ก่อนถึงจะเข้าหน้าแก้ไขได้
- กรอกชื่อผู้แก้ไขในหน้าต่าง Confirm → คลิก "Confirm" → ฟอร์มแก้ไขจะแสดง

---

### ❓ THF Form — ข้ามขั้นตอนได้หรือไม่?

สามารถคลิกที่หมายเลขขั้นตอน (Stepper) เพื่อข้ามไปยังขั้นตอนนั้นๆ ได้ แต่แนะนำให้กรอกข้อมูลตามลำดับและบันทึกทุกขั้นตอน
